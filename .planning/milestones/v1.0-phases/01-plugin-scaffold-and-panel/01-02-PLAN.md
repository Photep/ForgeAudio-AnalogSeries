**Document structure:**

### 1. Panel Dimensions
- Width: 60.96mm (12HP x 5.08mm/HP)
- Height: 128.5mm (standard 3U Eurorack)
- Panel center X: 30.48mm
- Screw clear zone: ~5mm diameter at each corner

### 2. Color Palette
| Name | Hex | Usage |
|------|-----|-------|
| Deep Navy | #1a1a2e | Panel background |
| Panel Border | #2d2d4a | Inset border stroke |
| Forge Amber | #e8a838 | Accent stripes, decorative lines, display border |
| Display Dark | #0d0d1a | Waveform display background |
| Muted Lavender | #8888aa | Brand text, jack labels |
| Light Lavender-Gray | #c0c0d0 | Control labels |
| Bright White-Gray | #e0e0e0 | Module name text |

### 3. Layout Zones (Diamond Hierarchy)
Table with zone name, Y range, height, and content description for each zone from top to bottom:
- Top accent + brand (0-8mm)
- Module name (8-14mm)
- Waveform display (17-42mm, ~25mm tall)
- Morph knob zone (46-60mm)
- Character/Drift zone (62-76mm)
- Rate/Octave zone (79-92mm)
- Section divider (94mm)
- Jack labels (97mm)
- Jack row (100-108mm)
- Bottom brand + accent (109-128.5mm)

### 4. Component Positions
Complete table with columns: Component, Widget Class, Center X (mm), Center Y (mm), Diameter (mm), Type (Param/Input/Output)

| Component | Widget Class | Center X | Center Y | Diameter | Type |
|-----------|-------------|----------|----------|----------|------|
| Morph | RoundBigBlackKnob | 30.48 | 52.0 | 15.24 | Param |
| Character | RoundLargeBlackKnob | 18.0 | 69.0 | 12.19 | Param |
| Drift | RoundLargeBlackKnob | 42.96 | 69.0 | 12.19 | Param |
| Rate | RoundBlackKnob | 18.0 | 86.0 | 9.60 | Param |
| Octave | RoundBlackKnob | 42.96 | 86.0 | 9.60 | Param |
| Morph CV | PJ301MPort | 11.0 | 104.0 | 8.03 | Input |
| Drift CV | PJ301MPort | 24.5 | 104.0 | 8.03 | Input |
| Output | PJ301MPort | 38.0 | 104.0 | 8.03 | Output |
| Inv. Output | PJ301MPort | 51.5 | 104.0 | 8.03 | Output |

### 5. SVG Component Layer Convention
Explain the hidden `<g id="components" style="display:none">` layer:
- Red circles = Params (knobs), radius = half component diameter
- Green circles = Inputs (jacks)
- Blue circles = Outputs (jacks)
- Circle center = widget placement center (matches mm2px coordinates in C++)

### 6. nanosvg Compatibility Constraints
Concise checklist of what nanosvg CANNOT render:
- NO `<text>` elements — convert all text to `<path>` elements
- NO CSS `<style>` blocks — use inline attributes (fill, stroke, etc.)
- NO `<use>`, `<clipPath>`, `<filter>`, `<image>` elements
- NO opacity on `<g>` groups — set on individual elements
- All colors as 6-digit hex (not 3-digit)
- Supported: `<rect>`, `<circle>`, `<ellipse>`, `<line>`, `<polyline>`, `<polygon>`, `<path>`, `<g>`
- Supported: Inline `fill`, `stroke`, `stroke-width`, `opacity`, `fill-opacity`, `stroke-opacity`
- Supported: 2-stop linear and radial gradients (since Rack 2.6.0)

### 7. Export Checklist
Step-by-step checklist for designers exporting from their tool:
1. Set document units to millimeters
2. Set width to 60.96mm, height to 128.5mm
3. Convert all text to curves/paths
4. Remove any CSS `<style>` blocks
5. Verify no `<text>` elements remain (search exported SVG)
6. Ensure viewBox matches: `viewBox="0 0 60.96 128.5"`
7. Verify `width="60.96mm"` and `height="128.5mm"` have mm suffix
8. Keep the hidden components layer intact (`<g id="components" style="display:none">`)
9. Do not change component circle positions (they must match C++ widget code)

### 8. Coordinate System Notes
- Origin (0,0) is top-left corner
- X increases rightward, Y increases downward
- All coordinates in millimeters
- Conversion to VCV Rack pixels: `pixels = mm * (75.0 / 25.4)` = `mm * 2.95276`
- The C++ code uses `mm2px(Vec(x, y))` which applies this conversion automatically

Include a note that the component positions in this spec, the SVG components layer, and the C++ widget code MUST stay synchronized. If a designer moves a component, the C++ code must be updated to match.
