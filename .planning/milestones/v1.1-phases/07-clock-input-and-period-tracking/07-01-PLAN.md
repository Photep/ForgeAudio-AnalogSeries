  <action>
Add the `processClockInput(float sampleTime)` method to the AnalogLFO struct (after the display update method, before the constructor) and call it from the top of `process()`.

**processClockInput() implements the complete three-state clock tracker per CONTEXT.md locked decisions:**

**Cable disconnect handling (top of method):**
- Check `inputs[CLK_INPUT].isConnected()`. If not connected and was previously connected (`prevClkConnected`):
  - Save `smoothedPeriod` to `lastSmoothedPeriod` (if > 0)
  - Set `clockState = FREE`, reset `clockEdgeCount = 0`, call `clockTimer.reset()`, set `smoothedPeriod = 0.f`
  - Store `displayClockState` as FREE (memory_order_relaxed)
- If not connected, set `prevClkConnected = false` and return early.
- Set `prevClkConnected = true`.

**Timer accumulation:**
- Call `clockTimer.process(sampleTime)` every sample.

**Timeout check (before edge detection, only when connected but no pulses):**
- If `clockState != FREE` and `smoothedPeriod > 0.f`:
  - Compute `timeout = std::fmax(1.0f, std::fmin(3.0f * smoothedPeriod, 5.0f))` (clamped 1s-5s per CONTEXT.md)
  - If `clockTimer.getTime() > timeout`: save `lastSmoothedPeriod`, transition to FREE, reset edge count, reset timer, update display atomic.

**Edge detection:**
- Get CLK voltage: `inputs[CLK_INPUT].getVoltage()`
- Call `clockTrigger.process(clkVoltage, 0.1f, 1.0f)` per CLK-02 (VCV standard thresholds)
- On trigger (returns true):
  - Read `rawPeriod = clockTimer.getTime()`, then `clockTimer.reset()`, increment `clockEdgeCount`
  - **Phase reset on every edge:** `phase = 0.0;` (hard reset, no crossfade -- per CONTEXT.md)

**First edge (clockEdgeCount == 1):**
- Enter ACQUIRING state. Update display atomic. No period measurement possible (only one edge so far).

**Second edge onward (clockEdgeCount >= 2, rawPeriod > 0.001f):**
- **Outlier rejection (LOCKED state only, per CLK-05):**
  - If LOCKED and `smoothedPeriod > 0.f`: check if `rawPeriod > 3.0f * smoothedPeriod` or `rawPeriod < smoothedPeriod / 3.0f`. If outlier, return early (silently discard).

- **Fast-track re-acquisition check:**
  - If state was FREE (before this edge promoted to ACQUIRING), `clockEdgeCount == 2`, and `lastSmoothedPeriod > 0.f`:
    - Compute `ratio = rawPeriod / lastSmoothedPeriod`
    - If `ratio > 0.8f && ratio < 1.2f`: fast-track to LOCKED. Set `smoothedPeriod = lastSmoothedPeriod`, apply one EMA update `smoothedPeriod += 0.3f * (rawPeriod - smoothedPeriod)`, set `clockState = LOCKED`, update display atomic, return.

  Note on state check: When `clockEdgeCount == 1` the state transitions to ACQUIRING. So at `clockEdgeCount == 2`, the state is ACQUIRING, not FREE. To implement fast-track, track whether the module was FREE before the current acquisition sequence. Use a boolean `wasReacquiring` or check `clockEdgeCount == 2` in ACQUIRING state AND `lastSmoothedPeriod > 0.f` as the fast-track condition. The key semantic: "this is the second edge of a fresh acquisition AND we have a remembered period to compare against."

- **EMA smoothing (per CLK-03):**
  - If `smoothedPeriod <= 0.f` (first valid measurement): snap directly `smoothedPeriod = rawPeriod` (avoids Pitfall 4 from research)
  - Otherwise: `smoothedPeriod += 0.3f * (rawPeriod - smoothedPeriod)` (alpha = 0.3)

- **State transition ACQUIRING -> LOCKED:**
  - If ACQUIRING and `clockEdgeCount >= 4`: transition to LOCKED, update display atomic. (4 edges = ~76% EMA convergence, per research recommendation)
  - Otherwise if ACQUIRING: update display atomic (keep ACQUIRING)

**Integration into process():**
- Add `processClockInput(args.sampleTime);` as the FIRST line inside `process()`, before the rate calculation.
- Do NOT change the frequency calculation. Phase 7 only stores `smoothedPeriod`; Phase 8 will use it. The LFO continues at Rate knob frequency at all times in Phase 7 (this is correct -- see research Open Question 3).

**Critical anti-patterns to avoid:**
- Do NOT override `freq` based on `smoothedPeriod` (that is Phase 8)
- Do NOT add crossfade/slew on phase reset (that is Phase 9)
- Do NOT serialize clock state (transient by design)
- Do NOT use integer sample counting (use dsp::Timer)
