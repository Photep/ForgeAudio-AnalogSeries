## Shared Patterns

### mm-anchor placement idiom (cross-cutting — every control + display box)
**Source:** the existing ctor, `src/AnalogLFO.cpp:1538–1585`
**Apply to:** display box, all 20 controls
```cpp
create*Centered<ForgeWidgetType>(mm2px(Vec(x_mm, y_mm)), module, AnalogLFO::ENUM)
```
Substitute only `x_mm`/`y_mm` from the LOCKED UI-SPEC table; never re-derive from the SVG (D-07, anti-pattern). `createParamCentered` handles the widget-size offset so the anchor is the *center* — no manual `box.pos` arithmetic.

### Widget-owns-the-knob rule (cross-cutting — D-01)
**Source:** `ForgeKnob*` structs `src/AnalogLFO.cpp:1451–1500` (each loads its own `_bg.svg`)
**Apply to:** the SVG strip and every knob anchor
One source of truth per knob: the panel SVG draws only surrounding chrome (sockets/labels/jack rings); the widget `_bg.svg` draws the body. Never let both draw the body.

### Proportional display layout (cross-cutting — display re-tune)
**Source:** `WaveformDisplay` `box.size.x * fraction` math, `src/AnalogLFO.cpp:908–913, 1321–1322`
**Apply to:** all display column/center math
Keep fraction-of-`box.size` math; it re-flows free. Only bump absolute-px constants (margins/fonts/insets). Reason in px or fractions inside the widget — `box.size` there is px (~308×98), never raw mm (Pitfall 5).

### Frozen surface (cross-cutting — zero regression)
**Apply to:** all edits
Do NOT touch `WaveformDisplay::step()`, `module->display*` atomics, RATIO_TABLE, swing math, DSP, param/enum bindings, or `plugin.json`. Only geometry/placement/SVG-art/widget-type-token change.
