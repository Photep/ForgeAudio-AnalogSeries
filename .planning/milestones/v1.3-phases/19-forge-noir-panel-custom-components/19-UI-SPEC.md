# Phase 19 -- UI Design Contract

> Visual and interaction contract for the Forge Noir 14HP panel and custom SVG components. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (VCV Rack native SVG pipeline) |
| Preset | not applicable |
| Component library | VCV Rack SDK: SvgKnob, SvgPort, SvgScrew base classes |
| Icon library | none -- all glyphs are SVG path elements |
| Font | All text as SVG `<path>` elements. Source fonts: FoundationLogo (brand), Bebas Neue (module name, hero/output labels), Chakra Petch (primary/small labels), JetBrains Mono (CV small labels) |

**Renderer constraints (nanosvg):**
- Maximum 2 stops per gradient (linear or radial)
- No `<text>`, `<style>`, `<use>`, `<filter>`, `<clipPath>`, `<image>` elements
- No CSS opacity on `<g>` groups -- set opacity on individual elements
- No 3-digit hex colors -- always use 6-digit (e.g., `#0c0c0c`)
- `<defs>` supported ONLY for gradient definitions
- All dimensions declared in mm with matching viewBox

## Spacing Scale

This phase uses mm coordinates derived from the forge-noir.html mockup at 5x scale. Conversion factor: `mm = px * (71.12 / 356) = px * 0.199775`.

| Token | Value (mm) | Value (5x px) | Usage |
|-------|-----------|---------------|-------|
| label-gap | 2.00mm | 10px | Gap between label bottom and component top edge |
| zone-gap | 2.00mm | 10px | Gap between component bottom and next label top |
| display-gap | 2.80mm | 14px | Gap between display bottom and first knob label |
| accent-bar | 0.60mm | 3px | Accent bar height at top/bottom panel edges |
| divider-inset | 5.60mm | 28px | Section divider inset from each panel side |
| cv-line-width | 0.20mm | 1px | CV connecting line width (trimpot to jack) |

Exceptions:
- Component positions are NOT grid-aligned -- they use exact mockup coordinates (see Component Position Map)
- Panel symmetry axis at x = 35.56mm (178px at 5x) is the governing alignment rule, not a spacing grid

## Typography

All text rendered as SVG `<path>` elements (nanosvg requirement per D-17). Font metrics below are authoring references for generating correct path data.

**4 declared sizes, 2 weights:**

| Role | Source Font | Size (5x px) | Size (mm) | Letter-spacing (5x px) | Color | Usage |
|------|------------|---------------|-----------|------------------------|-------|-------|
| Brand | FoundationLogo | 20px | 4.00mm | 5px | `#e85d26` | "FORGE" / "AUDIO" |
| Module Name | Bebas Neue | 20px | 4.00mm | 7px | `#e8e4e0` | "ANALOG LFO" |
| Hero/Output Label | Bebas Neue | 16px | 3.20mm | 5px | varies | MORPH (`#f0ece8`), OUTPUT (`#e85d26`) |
| Primary Label | Chakra Petch 600 | 9px | 1.80mm | 2.5px | `#c0bbb5` | CHARACTER, DRIFT |
| Small Label | Chakra Petch 500 / JetBrains Mono 500 | 7px | 1.40mm | 1.5px | varies | RATE, PHASE (`#888888`); CLK, RST (`#999999`); MORPH, CHAR, DRIFT, FM, PHASE CV row (`#777777`) |

**Size consolidation rationale:** The original design language specified 6 distinct sizes. Two merges reduce to 4 without perceptible loss at module scale: (1) Output Label (2.80mm, Bebas Neue) merged into Hero Label (3.20mm, Bebas Neue) -- same font family, 0.40mm difference imperceptible; (2) Secondary Label (1.60mm), I/O Label (1.60mm), and CV Label (1.40mm) unified at 1.40mm -- all serve the same "small supporting label" role and are differentiated by color, not size.

**Weights used:** Regular (400-500) for small labels, SemiBold (600) for primary labels. Brand and hero/output use display fonts where weight is inherent to the typeface design.

**Font-to-path conversion method:** Use Inkscape "Object to Path" for brand text (FoundationLogo). For smaller labels, the existing AnalogLFO.svg geometric block letter approach is acceptable. If Inkscape output is too complex, refine existing hand-crafted paths to match the target font's character.

## Component Rendering Contracts

### Knob Construction (all 3 sizes)

Each knob SVG (the rotating layer) contains:
1. **Knob body:** Circle filled with 2-stop radial gradient. Bright stop `#4a4a4a` at cx=38% cy=28% (off-center for specular highlight), dark stop `#0a0a0a` at r=100%.
2. **Center cap indent:** Smaller circle with 2-stop radial gradient. Bright stop `#282828` at cx=44% cy=38%, dark stop `#080808` at r=100%.
3. **Machined groove texture:** 3-4 concentric circle strokes at stroke-opacity 0.02 to 0.03, alternating `#ffffff` and `#000000`, simulating subtle lathe grooves.
4. **Indicator line:** Rect at 12-o'clock position (SvgKnob rotates it). Filled with 2-stop linear gradient from `#f08040` (top/outer) to `#c04020` (bottom/center). Width: 1mm for hero, 0.8mm for secondary, 0.6mm for utility. Height: 32% of diameter for hero, 28% for secondary, 26% for utility.
5. **Indicator glow halo:** Semi-transparent circle (r=1.2mm hero, 0.9mm secondary, 0.7mm utility) at the indicator tip, fill `#e85d26`, opacity 0.25. Adds premium glow effect within SVG constraints.

Each knob BG SVG (static behind knob) contains:
1. **Drop shadow:** Circle slightly larger than knob, fill `#000000`, opacity 0.4, offset 0.4mm downward.
2. **Metallic ring:** 4 concentric circles stepping inward: `#1a1a1a` (outer dark), `#4a4a4a` (bright ring), `#333333` (mid ring), `#1a1a1a` (inner dark). Ring width approximately 1.5mm total.

**Hero knob (MORPH) specific:** No knurled texture (D-05). Uses same machined concentric groove approach as other sizes. Includes morph arc: dashed circle (r=20.78mm at 5x = 104px diameter) centered on MORPH knob, stroke `#e85d26`, stroke-opacity 0.15, stroke-dasharray for dashed effect. The morph arc is baked into the panel SVG, not the knob SVG.

### Trimpot Construction

Trimpot SVG (rotating layer):
1. **Body:** Circle with 2-stop radial gradient. Bright stop `#6a6a6a`, dark stop `#1a1a1a`. Bright metallic appearance contrasting with dark knobs.
2. **Scalloped edge:** 8-10 small notch indents around the perimeter, approximated with small circles or arcs at low opacity to suggest finger grip.
3. **Indicator slot:** Narrow rect from center to edge at 12-o'clock. Linear gradient from `#cccccc` to `#888888`. Width 0.4mm, height 30% of diameter.

Trimpot BG SVG (static):
1. **Shadow ring:** Subtle dark circle, fill `#000000`, opacity 0.3.

Rotation range: minAngle = -0.75 * PI, maxAngle = 0.75 * PI (270 degrees).

### Jack Construction

**Input jack (36px / 7.19mm):**
1. **Flange:** Outermost circle, 2-stop radial gradient from `#555555` (center) to `#1a1a1a` (edge).
2. **Outer ring:** Circle stroke `#cccccc`, stroke-width 0.3mm, simulating metallic ring.
3. **Inner ring:** Second metallic circle, slightly smaller.
4. **Center hole:** Circle at 28% diameter, fill `#020202`, with inset shadow (darker circle slightly offset).
5. **Specular highlight:** Semi-transparent ellipse at upper-left, fill `#ffffff`, opacity 0.12.

**Output jack (42px / 8.39mm):**
- Same construction as input jack, scaled up.
- **Ember accent ring:** Additional circle stroke, `#e85d26`, stroke-width 0.6mm, stroke-opacity 0.5. Plus outer glow: circle stroke `#e85d26`, stroke-width 1.2mm, stroke-opacity 0.15. Both baked into the jack SVG per D-09.

### Hex Bolt Screw

1. **Outer hexagon:** 6-point polygon, fill with 2-stop linear gradient from `#333333` to `#1a1a1a` at 160 degrees.
2. **Inner hex socket:** Smaller hexagon, fill `#0a0a0a`, approximately 44% of outer diameter.

## Panel Decorative Elements

### Accent Bars (Top + Bottom)

- Height: 0.60mm (3px at 5x). Full panel width.
- Fill: 2-stop linear gradient. Since nanosvg supports only 2 stops, use left-to-right `#1a0800` to `#e85d26`. To approximate the HTML 5-stop gradient, layer two rects: one left-to-center gradient and one center-to-right gradient (mirrored).

### Header Slashes

- 3 diagonal slash marks on each side of the brand text header.
- Left side: rotated -25 degrees. Right side: rotated +25 degrees (mirrored about center axis).
- Each slash: rect approximately 0.4mm wide x 2.4mm tall.
- Fill: 2-stop linear gradient from `rgba(232,93,38,0.5)` at top to transparent at bottom.

### Module Name Decorative Line

- Thin horizontal line behind "ANALOG LFO" at its vertical center.
- 2-stop linear gradient from transparent (edges) to `rgba(232,93,38,0.25)` (center).
- The module name path elements sit on top, creating the symmetrical break visually.

### Section Dividers

- Horizontal gradient lines between knob sections.
- Inset 5.60mm (28px) from each panel side.
- 2-stop gradient approximation of the HTML ember-to-gold gradient.

### CV Connecting Lines

- 5 vertical lines, each linking a trimpot to its CV jack below.
- Width: 0.20mm (1px at 5x).
- 2-stop linear gradient from `rgba(232,93,38,0.25)` (top, at trimpot) to `rgba(232,93,38,0.08)` (bottom, at jack).

### Bottom Footer

- "FORGE" text rendered at opacity 0.25, positioned below the bottom I/O row. Subtle brand footer element.

## Forge Emblem Contract

Atmospheric symmetrical SVG behind the main knob section. All elements baked into panel SVG.

### Symmetry Method

Manual duplication (D-12 recommendation). For each left-side element at x_left, create a corresponding element at x_right = 71.12mm - x_left. Verify all pairs mirror perfectly about x = 35.56mm.

### Elements Per Side (mirrored)

1. **Molten metal streams:** 3 flowing path curves from center outward/downward. Widths: 3.20mm / 1.40mm / 0.50mm. Fill `#e85d26` at opacities 0.055 / 0.045 / 0.10.
2. **Forge marks flanking MORPH:** 2 chevron pairs pointing inward, 3 curved heat arc lines, 2 horizontal dash marks, 1 diamond accent. All at opacity 0.06-0.12.
3. **Ember particles:** ~11 small circles (r: 0.14-0.40mm) at opacities 0.08-0.18 in ember/gold/hot-gold colors.

### Centered Elements (inherently symmetric)

1. **Core forge glow:** Radial gradient ellipse behind MORPH. 2-stop gradient: `#e85d26` at opacity 0.20 (center) to opacity 0.00 (edge).
2. **Secondary warmth:** Larger gold radial gradient. `#daa520` at opacity 0.08 center to 0.00 edge.
3. **Central heat column:** Vertical rect, 5.60mm wide, opacity 0.03, `#e85d26`.
4. **Heat shimmer waves:** 3 horizontal sinusoidal path elements at different y-depths, `#e85d26`, opacity 0.02-0.04.

## Forge Rune Glyph Contract

Centered between "FORGE" and "AUDIO" brand text, vertically aligned with brand baseline.

### Elements (full detail per D-13 recommendation)

1. **Outer diamond:** 4-point path, stroke `#e85d26`, stroke-opacity 0.6, no fill.
2. **Inner diamond:** 4-point path, fill `#e85d26`, fill-opacity 0.18.
3. **Vertical axis:** Thin line through center with forked arrow tips at top and bottom.
4. **Horizontal extensions:** Short lines from diamond midpoints.
5. **Diagonal notch marks:** Small lines on diamond edges.
6. **Three-layer center glow:** Three concentric circles:
   - Outer: fill `#e85d26`, opacity 0.15
   - Mid: fill `#f0a030`, opacity 0.85
   - Core: fill `#ffe0a0`, opacity 0.90
7. **Radiating power rings:** 4 concentric circles behind the rune:
   - r=3.20mm, stroke-width 0.60mm, stroke `#e85d26`, opacity 0.14
   - r=4.00mm, stroke-width 0.40mm, stroke `#e85d26`, opacity 0.09
   - r=5.00mm, stroke-width 0.30mm, stroke `#e85d26`, opacity 0.06
   - r=6.20mm, stroke-width 0.20mm, stroke `#daa520`, opacity 0.035

No Gaussian blur (nanosvg constraint). Soften edges with wider low-opacity strokes instead.

## Validation Criteria

These map to the 6 checker dimensions, adapted for VCV Rack SVG panel context.

### Dimension 1: Copywriting
- All 10 label groups listed in Copywriting Contract are present as SVG path elements
- No `<text>` elements anywhere in any SVG file
- Letter-spacing matches DESIGN-LANGUAGE.md specifications

### Dimension 2: Visuals
- Panel loads in VCV Rack with no black rectangles or missing elements
- All 12 SVG component files render at correct scale (mm dimensions match spec)
- Knob indicator rotates with drag (not stuck, not reversed)
- Morph arc decorative element is visible around MORPH knob
- Forge emblem is visible but subtle behind knob section
- Rune glyph is visible between FORGE and AUDIO

### Dimension 3: Color
- Panel background is `#0c0c0c` (not the old `#1a1a2e`)
- Ember `#e85d26` appears only in the 12 reserved-for elements listed above
- Knob bodies use dark metallic gradients (not ember, not bright)
- Trimpot bodies are visually brighter than knob bodies
- Output jack has visible ember accent ring; input jacks do not

### Dimension 4: Typography
- Exactly 4 distinct font sizes used across all labels: 4.00mm (brand/module name), 3.20mm (hero/output labels), 1.80mm (primary labels), 1.40mm (small labels: secondary, CV, I/O)
- Size hierarchy is correct: Brand/Module Name (4.00mm) > Hero/Output Label (3.20mm) > Primary Label (1.80mm) > Small Label (1.40mm)
- Color hierarchy differentiates roles within same-size tiers: ember (brand/output) > warm white (module name/hero) > label light (primary) > label mid/I-O/dim (small labels)
- Maximum 2 font weights used: Regular (400-500) for small labels, SemiBold (600) for primary labels

### Dimension 5: Spacing
- 2.00mm (10px) gap between every label and its component
- Panel center axis symmetry: every element at x has a corresponding element at (71.12 - x) mm
- CV trimpots and jacks are vertically aligned in 5 equal-spaced columns
- Bottom I/O row (CLK, RST, OUTPUT) is evenly distributed across panel width

### Dimension 6: Registry Safety
- No external package dependencies
- All SVG files are hand-authored (no third-party SVG libraries)
- `minRackVersion` in plugin.json set to `"2.6.0"` to enforce gradient support
