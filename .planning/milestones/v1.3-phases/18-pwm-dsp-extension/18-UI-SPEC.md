## Waveform Display Behavior (Phase 18 Visual Contract)

Although no display code changes are needed, the following visual behaviors must hold true after DSP implementation. These are verification criteria, not implementation tasks:

### Pulse Shape Rendering

| Property | Expected Behavior |
|----------|-------------------|
| Morph 0.00-0.80 | Display shows sine, triangle, saw, square with crossfades -- identical to v1.2 behavior |
| Morph 0.80-1.00 | Display shows square transitioning to progressively narrower pulse |
| Morph 1.00 | Display shows narrow pulse at 5% duty cycle -- sharp spike with brief high and long low |
| Character 0 + pulse | Crisp digital pulse with vertical edges visible on display |
| Character 1 + pulse | Rounded analog pulse with tanh-softened edges visible as smooth curves on display |
| Bleed + pulse | At morph near 1.0 with character up, subtle sine contamination visible as slight waveform distortion |

### Waveform Continuity

| Transition | Expected Display |
|------------|-----------------|
| Square-to-pulse boundary (morph ~0.80) | Smooth crossfade visible -- no abrupt shape jump |
| Pulse-to-sine wrap (morph 1.0, bleed active) | Subtle neighbor mixing -- no visual discontinuity |
| Pulse at any morph + sweep character 0-to-1 | Edges progressively soften -- smooth visual transition |

### Amplitude Consistency

| Condition | Expected Display |
|-----------|-----------------|
| Narrow pulse (5% duty) at character 0 | Full amplitude -- pulse peak reaches +1, trough reaches -1 |
| Narrow pulse (5% duty) at character 1 | Near-full amplitude -- tanh softening may slightly reduce peak, but no visible amplitude collapse. Edge width scaling (`min(0.08, duty * 0.8)`) prevents the rounded bump failure mode. |

## Interaction Contract

### Morph Knob Behavior (extended)

| Knob Range | Shape | Visual on Display |
|------------|-------|-------------------|
| 0.00 | Pure sine | Smooth sinusoidal curve |
| 0.00-0.20 | Sine-to-triangle crossfade | Sinusoid corners progressively sharpen |
| 0.20 | Pure triangle | Symmetric pointed waveform |
| 0.20-0.40 | Triangle-to-saw crossfade | Asymmetry increases, one edge steepens |
| 0.40 | Pure saw | Ramp up with sharp reset |
| 0.40-0.60 | Saw-to-square crossfade | Top and bottom flatten toward rectangular |
| 0.60 | Pure square | 50% duty rectangular wave |
| 0.60-0.80 | Square-to-pulse crossfade | Positive region narrows, negative region widens |
| 0.80 | Pulse at 50% duty (matches square) | Identical to square appearance |
| 0.80-1.00 | Pulse narrowing | Positive spike becomes progressively thinner |
| 1.00 | Narrow pulse at 5% duty | Brief sharp spike, long flat trough |

**Source:** D-01 (even 5-shape rescale), D-03 (morph order), D-07 (duty range), D-08 (linear duty mapping)

### Character Knob Interaction with Pulse

| Character | Effect on Pulse | Visual |
|-----------|----------------|--------|
| 0.00 | Digital pulse -- crisp vertical edges | Sharp rectangular spike |
| 0.25 | Slight edge softening via tanh | Very slightly rounded corners |
| 0.50 | Moderate analog character | Noticeably rounded edges, subtle asymmetry from spread |
| 0.75 | Strong analog character | Smooth bell-like pulse, visible edge softening |
| 1.00 | Full analog deformation | Heavily rounded pulse approaching Gaussian bump shape |

**Source:** D-05 (tanh edge softening), D-06 (spread affects edges only, not duty cycle)

## Verification Checklist (for UI auditor)

These visual behaviors can be verified by loading the built module in VCV Rack and observing the waveform display:

- [ ] Sweep morph 0-to-1: five distinct shapes visible in sequence (sine, tri, saw, square, pulse)
- [ ] At morph=1.0: narrow pulse spike visible, clearly different from square wave
- [ ] Sweep morph through square-to-pulse boundary (~0.60-0.80): no visual discontinuity or jump
- [ ] At morph=1.0, sweep character 0-to-1: pulse edges visibly soften
- [ ] At morph=1.0, character=1.0: pulse is rounded but still reaches near-full amplitude (no collapsed bump)
- [ ] At morph=0.0 to 0.60: waveform display behavior identical to v1.2 (verify sine, tri, saw, square unchanged)
- [ ] Display render frequency is smooth (no visual jitter from pulse computation)
