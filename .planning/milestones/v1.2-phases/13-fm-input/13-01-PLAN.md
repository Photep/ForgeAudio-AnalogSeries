NOTE: Default is 0.0 (NOT 1.0 like other attenuators). FM is opt-in: patching a cable should NOT immediately modulate until user turns up the attenuator. This is critical for success criterion 4 (identical v1.1 behavior at zero).

**3. FM processing block** -- Insert in process() between the targetFreq calculation (line ~480) and the existing clamp (line 481 `targetFreq = std::fmax(targetFreq, 0.001f)`). The FM block goes BEFORE the clamp and slew:
```cpp
// FM processing (MOD-01, MOD-02)
if (inputs[FM_INPUT].isConnected()) {
    float fmCV = inputs[FM_INPUT].getVoltage();
    float fmAtten = params[FM_ATTEN_PARAM].getValue();
    // Authority: reduced in clocked mode to prevent clock-phase fighting (MOD-02)
    float depthScale = isClocked ? 0.1f : 0.6f;
    float fmPitch = fmCV * fmAtten * depthScale;
    targetFreq *= dsp::exp2_taylor5(fmPitch);
}
```
The isConnected() guard avoids wasting CPU on exp2 when no cable is patched (follows existing PHASE_OFFSET_CV_INPUT pattern at line 553).
