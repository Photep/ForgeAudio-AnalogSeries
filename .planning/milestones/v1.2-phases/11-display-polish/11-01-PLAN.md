In the WaveformDisplay struct in src/AnalogLFO.cpp, create a new `drawPillText()` method that replaces `drawGlowText()` for all overlay text. This method:

1. Sets NanoVG font state (face, size, alignment) for measurement
2. Measures text bounds with `nvgTextBounds(vg, x, y, text, NULL, bounds)`
3. Calculates pill dimensions: text bounds + 4px padding on all sides
4. Draws a feathered pill background using `nvgBoxGradient`:
   - Inner color: `nvgRGBAf(0.102f, 0.102f, 0.180f, 0.80f * alpha)` (#1a1a2e at 80% opacity, scaled by alpha)
   - Outer color: `nvgRGBAf(0.102f, 0.102f, 0.180f, 0.0f)` (fade to transparent)
   - Corner radius: 3px
   - Feather distance: 3px
   - CRITICAL: The `nvgRoundedRect` path must be larger than the box gradient bounds by the feather distance on all sides, otherwise feathered edges get clipped. Path = `(px - feather, py - feather, pw + 2*feather, ph + 2*feather, radius + feather)`
   - Use `nvgFillPaint(vg, pillPaint)` — NOT `nvgFillColor` (solid fills have hard edges)
5. Draws the existing 2-pass glow text on top:
   - Pass 1: blur=3.0, amber at alpha*0.4
   - Pass 2: blur=0.0, amber at full alpha

Signature: `void drawPillText(NVGcontext* vg, int fontHandle, float x, float y, const char* text, float fontSize, int align, float alpha)`

Then update `drawTextOverlays()`:
- Replace all 4 `drawGlowText()` calls with `drawPillText()` calls (Hz readout, ratio label, SYNC badge, BPM readout)
- Same arguments, same positions — only the function name changes
- Keep `drawGlowText()` as-is (don't delete it — may be used elsewhere or in future)

Per user decisions:

- All text stays amber (0.91, 0.66, 0.22)
- Keep existing 2-pass glow rendering
- Free-running mode: Hz readout gets pill background (no other changes)
- Clocked mode: ratio, SYNC, BPM all get pill backgrounds

<task type="auto">
  <name>Task 2: Add dual-line BPM stack with CLK/BPM display and x1 collapsing (DISP-02)</name>
  <files>src/AnalogLFO.cpp</files>
  <action>
In the WaveformDisplay struct, create a new `drawBpmStack()` method and update the BPM section of `drawTextOverlays()` to use it. This replaces the single-line BPM `drawPillText()` call (from Task 1) with a dual-line stack.

`drawBpmStack()` method:

1. Read display state:
   - `ratioIdx` from `module->displayRatioIndex` (relaxed atomic)
   - `period` from `module->displaySmoothedPeriod` (relaxed atomic)
   - Guard: return early if ratioIdx < 0 or period <= 0

2. Calculate BPM values:
   - `rawBPM = 60.f / period` (raw incoming clock tempo)
   - `effectiveBPM = rawBPM * AnalogLFO::RATIO_TABLE[ratioIdx]` (ratio-adjusted)
   - `isX1 = (ratioIdx == 7)` (RATIO_TABLE[7] = 1.0)

3. Format text strings:
   - BPM formatting: `"%d BPM"` for >= 1 BPM, `"%.1f BPM"` for < 1 BPM
   - CLK formatting: `"%d CLK"` for >= 1, `"%.1f CLK"` for < 1

4. x1 ratio (single line): Call `drawPillText()` with effective BPM text at bottom-right (`box.size.x - margin`, `box.size.y - margin`), font size 10px, right-bottom aligned, using the passed-in alpha.

5. Non-x1 ratio (dual line with shared pill):
   - Effective BPM line: 10px font, full alpha, positioned at bottom-right
   - CLK line: 8px font, 60% alpha (dimmed), positioned above BPM line
   - Vertical spacing: Use `nvgTextMetrics()` to get BPM line height, place CLK line above with 2px gap
   - Shared pill: Measure both lines' text bounds with `nvgTextBounds()`, union them into a single bounding rect, add 4px padding, draw one feathered pill background covering both lines
   - Then draw CLK glow text (2-pass, 60% alpha on both blur and sharp passes) and BPM glow text (2-pass, full alpha) on top of the shared pill
   - The CLK line alpha = `alpha * 0.6f` (dimmed relative to fade state)
   - The CLK glow pass alpha = `alpha * 0.6f * 0.4f` (proportionally dimmed glow)

6. Update `drawTextOverlays()`: Replace the existing BPM block (the `if (bpmFadeAlpha > 0.001f ...)` section) with a call to `drawBpmStack(vg, font->handle, bpmFadeAlpha)`.

Per user decisions:
- BPM stack appears immediately during ACQUIRING state (bpmFadeAlpha already fades in when clocked)
- Both CLK and BPM lines share one pill background
- CLK line is supplemental context (smaller, dimmer)
- At x1: only "120 BPM" shown, no CLK line
- At non-x1: "120 CLK" above "30 BPM" (for example at /2 ratio)
