## Tech Debt

### Phase 2: Waveform Engine
- **OUT-02 documentation inconsistency:** REQUIREMENTS.md body text still reads "Inverted morphed waveform output" but the INV output was removed by user design decision (documented in 02-CONTEXT.md). The traceability table correctly marks OUT-02 as Complete. Body text should be updated to reflect the removal.

### Phase 5: Drift Engine
- **Drift dot instability too subtle:** User noted dot instability visual is "not noticeable" at current parameters (0.3f trail jitter, 0.15f halo variation). Accepted for v1; cosmetic tuning could increase these values.
- **Panel bottom row design language:** 7-component bottom row uses non-standard grouping. User noted this during verification. Cosmetic refinement opportunity.
- **Dot instability reads knob only:** `drawPhaseDot()` reads `DRIFT_PARAM` directly rather than the CV-modulated drift value computed in `process()`. This means trail jitter and halo variation don't respond to Drift CV. The dot *speed* (via `displayPhase`) correctly responds to CV-modulated drift. Cosmetic inconsistency only.

**Total:** 4 items across 2 phases. None are functional blockers.
