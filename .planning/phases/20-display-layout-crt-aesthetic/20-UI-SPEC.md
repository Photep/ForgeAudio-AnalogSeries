## Spacing Scale

Context: This phase operates within a 188.8 x 53.1 pixel NanoVG widget. Standard 8-point spacing is adapted to the small display scale. All values are widget-pixel measurements.

| Token | Value | Usage |
|-------|-------|-------|
| inset | 3px | Corner bracket inset from display edge, pill inset from column edge |
| gap-xs | 2px | Padding inside pills around text |
| gap-sm | 3px | Pill internal padding |
| gap-md | 5px | Bracket arm length, column gutter between pill columns and center |
| margin | 4px | General display edge margin (preserved from current code) |
| col-left | 0-20% | Left pill column (0px to ~37.8px of 188.8px width) |
| col-center | 20-80% | Center waveform column (~37.8px to ~151.0px) |
| col-right | 80-100% | Right pill column (~154.6px to ~188.8px) |

Column boundary constants (as fractions of `box.size.x`):

| Constant | Value | Widget px (at 188.8px) |
|----------|-------|------------------------|
| LEFT_COL_START | 0.05 | 9.4px |
| LEFT_COL_END | 0.17 | 32.4px |
| CENTER_START | 0.20 | 37.8px |
| CENTER_END | 0.80 | 151.0px |
| RIGHT_COL_START | 0.82 | 154.6px |
| RIGHT_COL_END | 0.95 | 178.2px |

Exceptions: All spacing is proportional to `box.size.x` / `box.size.y`, not hardcoded pixel values, to support VCV Rack zoom levels. Source: RESEARCH.md Pitfall 5.

## Typography

Font: JetBrains Mono NL Regular (`res/fonts/JetBrainsMonoNL-Regular.ttf`)

Loaded per frame via `APP->window->loadFont(asset::plugin(pluginInstance, "res/fonts/JetBrainsMonoNL-Regular.ttf"))`. Never cached across frames (VCV Rack GPU context can be destroyed and recreated). Source: RESEARCH.md anti-patterns.

| Role | Size (widget px) | Size (mockup px) | Weight | Line Height | Usage |
|------|-------------------|-------------------|--------|-------------|-------|
| Pill Label | 4.1px | 7px | 400 | 1.0 (single-line) | Ratio display, SYNC badge text |
| Pill Value | 3.5px | 6px | 400 | 1.0 (single-line) | BPM value, CLK/LFO labels |
| Pill Small | 3.2px | 5.5px | 400 | 1.0 (single-line) | Swing percentage text |
| Pill Micro | 2.9px | 5px | 400 | 1.0 (single-line) | Hz readout, SWING label, CLK/LFO sub-labels |

Source: RESEARCH.md Dimension Reference > Font Sizes. CONTEXT.md D-07 mandates JetBrains Mono. Claude's discretion on weight/size tuning for readability at small display size.

**Readability rule:** If any text renders as unreadable blobs at 100% VCV Rack zoom, bump font size up by 0.5px increments until legible. Readability trumps exact mockup fidelity at sub-4px sizes. Source: RESEARCH.md Pitfall 3.

## Copywriting Contract

This phase has no user-facing text input, no CTAs, no empty states, and no error states in the traditional sense. All "text" is real-time data readout rendered via NanoVG within the display widget. The copywriting contract specifies the exact text format for each data pill.

| Element | Format | Example | Position |
|---------|--------|---------|----------|
| Ratio pill | `x{ratio}` or ratio fraction | `x2`, `x1/2`, `x1` | Left column, top |
| Hz readout | `{value} Hz` (1 decimal if <10, integer otherwise) | `2.4 Hz`, `120 Hz` | Left column, below ratio |
| Swing pill | `{percentage}%` | `67%` | Left column, bottom |
| SWING label | `SWING` (static) | `SWING` | Left column, below swing value |
| SYNC badge | `SYNC` (static) | `SYNC` | Right column, top |
| CLK label | `CLK` (static) | `CLK` | Right column, BPM stack top line |
| LFO label | `LFO` (static) | `LFO` | Right column, BPM stack top line (alternative) |
| BPM value | `{value} BPM` (integer if >=1, 1 decimal otherwise) | `120 BPM`, `0.5 BPM` | Right column, BPM stack bottom line |

### Display States

| State | What Renders | Condition |
|-------|-------------|-----------|
| Free-running (default) | Hz readout visible, ratio/SYNC/BPM/swing hidden | No clock input connected |
| Clocked | Ratio + SYNC + BPM visible, Hz hidden, swing visible if >Straight | Clock input connected and locked |
| Placeholder (no module) | Static sine waveform, no pills | Module browser preview (module == nullptr) |

### Fade Transitions

All pill visibility transitions use 200ms linear fade (existing `fadeSpeed = 1.f / (0.2f * 60.f)` pattern). Source: existing code line 868.

## Component Inventory

All components are NanoVG draw calls within the WaveformDisplay struct. No external component library.

### New Draw Methods (Phase 20)

| Method | Purpose | Replaces |
|--------|---------|----------|
| `drawBorder()` | 1.5px ember border + breathing glow | `drawInsetFrame()` |
| `drawCornerBrackets()` | Four L-shaped ember brackets at corners | Nothing (new) |
| `drawZeroCrossing()` | Dashed center reference line in center column | Nothing (new) |
| `drawScanlines()` | CRT scanline overlay with slow scroll | Nothing (new) |

### Modified Draw Methods

| Method | Change | Source |
|--------|--------|--------|
| `drawBackground()` | Fill color #0d0d1a to #030303 | CONTEXT.md D-12 |
| `drawWaveformTrace()` | 4-pass glow to 3-layer glow, ember color, center column constraint | CONTEXT.md D-09/D-11, RESEARCH.md Pattern 1 |
| `drawPhaseDot()` | Radial gradient to 3-circle concentric, gold core | CONTEXT.md D-11, RESEARCH.md |
| `drawPillText()` | Navy feathered box to ember-tinted fill + stroke | CONTEXT.md D-06/D-08 |
| `drawTextOverlays()` | Corner-based positions to column-based positions | CONTEXT.md D-01/D-03 |
| `drawBpmStack()` | Position to right column, ember pill styling | CONTEXT.md D-01 |
| `phaseToX()` | Full-width to center-column mapping (20%-80%) | CONTEXT.md D-02, RESEARCH.md Pattern 1 |
| `step()` | breathePhase rate 0.8Hz to 0.2Hz | CONTEXT.md D-13, RESEARCH.md Pitfall 2 |

### New Struct Members

| Member | Type | Purpose |
|--------|------|---------|
| `scanlineImage` | int (-1 initial) | NanoVG image handle for repeating scanline tile. Lazy-created in first drawScanlines() call. |
| `scanlineScrollPhase` | float (0.f initial) | Accumulator for scanline scroll offset. Wrapped with fmodf(, 4.f) each frame. |

### Rendering Order (drawLayer sequence)

Source: RESEARCH.md Architecture Patterns.

```
drawLayer(layer 1):
  1. drawBackground()        -- #030303 fill
  2. drawBorder()            -- 1.5px ember stroke + breathing glow  [replaces drawInsetFrame]
  3. drawCornerBrackets()    -- 4 L-shaped ember brackets            [NEW]
  4. drawZeroCrossing()      -- dashed center line in center column  [NEW]
  5. drawWaveformTrace()     -- 3-layer glow, center column only     [MODIFIED]
  6. drawPhaseDot()          -- dot + comet trail, center column     [MODIFIED colors]
  7. drawTextOverlays()      -- pills in left/right columns          [MODIFIED positions+style]
  8. drawScanlines()         -- subtle scrolling overlay on top      [NEW]
```
