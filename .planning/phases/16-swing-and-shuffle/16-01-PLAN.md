<objective>
Add MPC-style swing timing to the clocked LFO (PHASE-03, PHASE-04). Swing modifies the deltaPhase accumulation rate so the first half of each cycle (phase 0-0.5) plays slower and the second half (0.5-1.0) plays faster, creating the classic long-short beat pair groove. Controlled via right-click context menu with 6 named presets. Display shows swing-warped waveform and text overlay. Swing is gated by `isClocked` -- no effect in free-running mode. At 50% (Straight), behavior is identical to v1.1.

Purpose: Swing timing transforms the LFO from a straight metronome into a groove instrument. Essential for rhythmic modulation patches in electronic music production.

Output: Modified src/AnalogLFO.cpp with swing state, audio-path deltaPhase multiplier, right-click context menu, serialization, phase-warped display buffer, swing text overlay, and swing-aware phase dot positioning.

**2. Modify updateDisplayBuffer() to accept swing fraction (line 313):**

Change the function signature and body:
```cpp
void updateDisplayBuffer(float morph, float character, float swingFrac = 0.5f) {
    int writeIdx = 1 - displayReadIdx.load(std::memory_order_relaxed);
    for (int i = 0; i < DISPLAY_SAMPLES; i++) {
        float t = (float)i / (float)DISPLAY_SAMPLES;  // uniform time
        float p;
        if (swingFrac <= 0.5001f) {
            p = t;  // fast path: no swing
        } else if (t < swingFrac) {
            p = t * 0.5f / swingFrac;                              // even: [0,S) -> [0,0.5)
        } else {
            p = 0.5f + (t - swingFrac) * 0.5f / (1.f - swingFrac); // odd: [S,1) -> [0.5,1)
        }
        displayBuffers[writeIdx][i] = computeMorphedWave(p, morph, character);
    }
    displayReadIdx.store(writeIdx, std::memory_order_release);
}
```

The default parameter `swingFrac = 0.5f` ensures backward compatibility -- any existing call site that doesn't pass swingFrac gets identity behavior. The x-axis represents uniform TIME; each sample's phase lookup is warped by the inverse swing function. At S=0.5 this simplifies to `p = t` (identity). At S=0.66 (triplet), the first 2/3 of display width shows phase 0-0.5, and the last 1/3 shows phase 0.5-1.0.

**Key design notes for the executor:**
- The swing multiplier is applied AFTER drift/jitter but BEFORE `phase += deltaPhase`. Order doesn't matter mathematically (multiplication is commutative), but this placement follows the conceptual model: drift perturbs the swing-structured timing.
- NO smoothing at the phase 0.5 boundary. The instant rate change is intentional -- it matches MPC/Akai swing behavior. Smoothing would dilute the groove feel.
- NO special cases for different ratios. Swing always warps the individual LFO cycle regardless of ratio (x1, x2, /4, etc.). This was a resolved STATE.md blocker.
- Swing and Phase Offset are independent layers. Phase Offset applies at readout (line 704), swing applies to deltaPhase. They stack naturally.

Do NOT modify:
- computeMorphedWave() -- swing is a timing phenomenon, not a waveshaping change
- processClockInput() -- swing does not affect clock tracking
- processResetInput() -- swing does not affect reset behavior
- The four compute functions (computeSine, computeTriangle, computeSaw, computeSquare)
- Any Phase Offset logic -- swing and Phase Offset are independent
- Any drift/jitter logic -- swing multiplies deltaPhase alongside drift (commutative)
