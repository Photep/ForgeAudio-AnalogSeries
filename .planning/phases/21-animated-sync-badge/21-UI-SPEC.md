## Color

The badge is an **accent element** in the Forge Noir palette. Steady state is full-alpha ember. The flash is the only new color behavior in this phase.

| Role | Value (NanoVG `nvgRGBAf`) | Hex | Usage |
|------|---------------------------|-----|-------|
| Display background (dominant) | `0.012, 0.012, 0.012` | ~#030303 | Display surface (unchanged) |
| Ember accent (steady state) | `0.91, 0.365, 0.149` | ~#e85d26 | SYNC text, glow, pill fill/stroke — steady-state badge color |
| White-hot peak target | `1.0, 0.93, 0.78` (recommended; tunable at gate) | ~#ffeec7 | The hue the badge **lerps toward at flash peak** (warm white-yellow, not pure white — keeps "molten metal" read) |

**Accent reserved for:** the SYNC badge text, its glow pass, and its pill fill/stroke. The flash must NOT bleed onto neighboring elements (BPM stack, ratio pill, border glow). No new color is introduced — the peak is a lerp between the two ember/white-hot endpoints already named.

**Destructive color:** N/A — reason: no destructive actions exist in a passive status badge.

### Color/Glow Flash Contract (the core of this phase)

The flash is driven by a single normalized envelope value `flash ∈ [0, 1]` (1 = just-triggered peak, decaying toward 0 = steady state). All flash-driven visuals are functions of `flash`:

| Channel | Steady state (`flash = 0`) | Peak (`flash = 1`) | Mapping |
|---------|----------------------------|--------------------|---------|
| Text color | ember `(0.91, 0.365, 0.149)` | white-hot `(1.0, 0.93, 0.78)` (tunable) | `lerp(ember, whiteHot, flash)` on sharp pass AND glow pass |
| Glow blur radius | `2.0` (existing `nvgFontBlur`) | `~4.0–5.0` (tunable at gate) | `lerp(2.0, peakBlur, flash)` — the bloom |
| Glow alpha multiplier | `0.3 × alpha` (existing) | `~0.6–0.9 × alpha` (tunable) | `lerp(0.3, peakGlowAlpha, flash) × alpha` — stronger bloom |
| Sharp-pass alpha | `1.0 × alpha` (full) | `1.0 × alpha` (stays full) | brightness comes from **color + glow**, NOT alpha (D-01: steady state is already full alpha) |
| Pill fill/stroke | ember at 0.10 / 0.25 × alpha | unchanged (recommended) | flash drives **text + glow only**; pill background stays calm so the text "strikes" against it |

Exact peak constants (`whiteHot` hue, `peakBlur`, `peakGlowAlpha`) are a **single-cluster tune finalized at the acceptance gate** (D-03). The contract fixes the *mechanism* and *endpoints*; the gate fixes the *magnitude*.

## Animation Contract

The defining contract of Phase 21. Expressed as states + an envelope + a timing curve.

### Badge States (mutually exclusive, keyed on `clockState`)

| Clock state | Badge behavior | Driver | Status |
|-------------|----------------|--------|--------|
| FREE (unclocked) | Badge faded out (`syncFadeAlpha → 0`); not drawn below 0.001 | existing `syncFadeAlpha` | unchanged |
| ACQUIRING | Steady ember + **2 Hz blink** (`blink = 0.5 + 0.5·sin(blinkPhase)`) | existing `blinkPhase` | unchanged — D-02 keeps blink here |
| LOCKED | Steady ember + **per-edge white-hot flash** with exponential decay | NEW `flash` envelope | this phase |

**D-02 clean split (LOCKED contract):** 2 Hz blink = "acquiring / searching"; per-edge flash = "locked & responding to each beat." The two animations never run simultaneously on the badge. While LOCKED, no blink is applied; while ACQUIRING, no flash is applied.

### Flash Trigger

| Property | Contract |
|----------|----------|
| Trigger event | Each detected clock edge **while `clockState == LOCKED`** |
| DSP source | `clockTrigger.process(...)` site (`src/AnalogLFO.cpp:446`); increment a lock-free edge signal only when LOCKED |
| Audio→widget bridge | `std::atomic<int>` edge counter (recommended), mirroring existing `display*` atomics (lines 110–162); widget's `step()` compares against a cached last-seen count |
| On new edge | Set `flash = 1.0` (re-arm to peak; re-triggers cleanly even mid-decay) |

Exact field names/shape are the planner's call (delegated in CONTEXT.md). The contract only fixes: lock-free, LOCKED-gated, re-armable to peak on each edge.

### Decay Curve (LOCKED by ANIM-02 — do not propose an alternative)

| Property | Contract |
|----------|----------|
| Model | **Exponential decay**, `flash *= 0.92` per frame in `WaveformDisplay::step()` |
| Per-frame factor | `~0.92` (locked by ANIM-02) |
| Effective duration | `~200 ms` to return to visually steady state (≈ Phase 20's `1/(0.2·60)` fade convention; ~12 frames at 60 fps to ~0.37, ~0.05 by ~36 frames) |
| Floor | Clamp/treat `flash` as 0 below a small epsilon (e.g. `< 0.001`) to stop animating and render exact steady state |
| Feel | Smooth, non-strobe — "spark striking hot metal, cooling back to ember." Success criterion #2: no abrupt transitions |
| Advance location | `WaveformDisplay::step()` (`src/AnalogLFO.cpp:868`), alongside existing `breathePhase` / `blinkPhase` / fade timers |

### Intensity (D-03)

Peak must be **clearly visible / punchy** — distinct from steady-state ember at a glance (success criterion #1), driven by color shift + glow bloom together. Restrained Forge Noir character is preserved by the smooth exponential decay (no harshness). Exact peak magnitude is tuned at the acceptance gate.

### Interaction / accessibility dimensions

- **Focus / keyboard states:** N/A — the badge is a passive status indicator, not an interactive control.
- **Reduced-motion preference:** N/A — VCV Rack has no per-widget reduced-motion API; the decay is already smooth and non-strobing (no rapid flicker), satisfying the restraint intent.
- **Responsive breakpoints:** N/A — fixed-size 18HP rack panel; scales uniformly with Rack zoom only.
