# Phase 20.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
>
> **Surface reality:** This is a VCV Rack 2 C++ faceplate phase (precise repositioning + artwork-swap). There is **no headless geometry-test harness** for VCV faceplates — validation is structured as a **compile gate + in-Rack alignment checklist + Phase-20 regression diff** (D-08). The only automated signal is "it compiles"; every fidelity criterion is a manual in-Rack visual check at 100% and 200% zoom.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | None automated — VCV Rack has no headless geometry test rig. Validation = compile gate + manual in-Rack visual inspection. |
| **Config file** | `Makefile` (RACK_DIR=../Rack-SDK) |
| **Quick run command** | `make` (compile gate — must build with zero errors) |
| **Full suite command** | `make clean && make install` (clean rebuild + install — stale `dist/` and installed plugin dir cache old art) |
| **Estimated runtime** | ~30–90 seconds (compile) |

---

## Sampling Rate

- **After every task commit:** Run `make` — never commit a non-compiling ctor.
- **After every plan wave:** Run `make clean && make install`, then re-check the alignment checklist below.
- **Before `/gsd:verify-work`:** Clean reinstall green + full alignment checklist confirmed at 100% AND 200% + screenshots captured.
- **Max feedback latency:** ~90 seconds (compile); visual gate is human-paced.
