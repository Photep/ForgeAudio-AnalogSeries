# Phase 20.1: Panel Redesign 18HP Fresh Layout - Context

**Gathered:** 2026-06-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Ship the Analog LFO on the new 18HP (91.44mm) Forge Noir "fresh" panel. Every knob, trimpot, input jack, the output jack, and the waveform display is repositioned to sit exactly on its new artwork anchor (anchors locked in ROADMAP.md), the four secondary knobs become an equal-sized single row (CHARACTER/DRIFT/RATE/PHASE, with RATE/PHASE upgraded from the smaller utility size), and the display refits the larger 81.44×26.00mm box — all with **zero regression** to Phase 20 display content/polish (ember waveform, scanlines, breathing glow, corner brackets, three-column layout) and no loss of existing functionality (params, CV attenuverters, CLK/RST inputs, OUTPUT).

This is a precise repositioning + artwork-swap phase. Anchors, requirements (PANEL-08…12), and success criteria are already locked by the ROADMAP. Discussion below clarifies HOW to implement, not WHAT to build.

</domain>

<decisions>
## Implementation Decisions

### Knob Rendering vs Panel Artwork
- **D-01:** The fresh panel SVG currently draws full static metal knob graphics (bezel + inner ring + center) at all 10 knob anchors. The current 14HP panel drew none — the widget knobs (`SvgKnob` subclasses with their own `_bg.svg` body) own the entire knob. **Keep that proven pattern.** Strip the metal knob graphics from the fresh SVG so the widget owns the full knob — no double-rendered knob bodies. A soft shadow/anchor ring under each knob is acceptable to keep if it reads well, but the metal body/inner-ring/center art comes out.
- **D-02:** RATE and PHASE upgrade from `ForgeKnobUtility` to `ForgeKnobSecondary` so all four secondary knobs are visually equal (CHARACTER, DRIFT, RATE, PHASE) in the single y=87 row. MORPH stays `ForgeKnobHero`.

### Panel Artwork Promotion (Cleanup)
- **D-03:** Promote `res/AnalogLFO-fresh.svg` → `res/AnalogLFO.svg` (overwrite the old 14HP file). The production panel path stays `res/AnalogLFO.svg`, so `setPanel(createPanel(... "res/AnalogLFO.svg"))` in the widget ctor is **unchanged** — only the file content changes (to the bezel-stripped 18HP art per D-01).
- **D-04:** Remove the now-dead artwork: `res/AnalogLFO-redesign.svg` and `res/AnalogLFO-fresh-v1.svg`. fresh.svg supersedes them. Remove any dead 14HP position constants/comments left in `AnalogLFOWidget` after repositioning.
- **D-05:** No `plugin.json` / manifest width change needed — VCV Rack derives module HP/width from the panel SVG's dimensions automatically.

### Display Refit (larger box)
- **D-06:** Display box moves/resizes to pos (5.00, 19.00), size (81.44, 26.00) mm — ~27% wider, ~45% taller than the Phase 20 box (63.93×17.98). All Phase 20 three-column content is preserved with no text clipping or overflow. The SYNC badge stays in the display's right column (Phase 20 D-01) — Phase 21 depends on it being in its final position here.

### Anchor Fidelity & Verification
- **D-07:** Repositioning uses the locked mm anchor table from ROADMAP.md, applied via the existing `mm2px(Vec(x, y))` / `createParamCentered` / `createInputCentered` style already in the widget ctor (hardcoded mm constants matching the current code idiom — no SVG parsing required).
- **D-08:** Verification is an **in-Rack visual check**: build the plugin, load the module in VCV Rack, and eyeball every control/jack/display against the artwork at both 100% and 200% zoom, capturing screenshots as proof. The phase is not done until alignment is confirmed at both zoom levels with no visible offset and no Phase 20 display regression.

### Hex Bolts
- **D-09:** Hex bolt corners are at (4.60,4.60), (86.84,4.60), (4.60,123.90), (86.84,123.90) — derive horizontal placement from the new 18HP box width (current ctor uses `box.size.x` math for the right-side bolts, which adapts automatically once the panel width changes).

### Claude's Discretion
- **Display content scaling:** How the Phase 20 three-column content fills the larger glass is the implementer's call. Pick whatever reads best at actual widget scale — scale pills/fonts/waveform up, or keep sizes and let the extra space become margin/headroom — as long as Phase 20 content/polish is preserved with no clipping or overflow. Re-tune the internal layout constants (`phaseToX`/`valueToY` center-column math, pill column x-fractions, BPM stack, font sizes) against the new 81.44×26.00mm box.
- **Soft shadow/anchor retention:** Whether to keep a subtle under-knob shadow in the stripped SVG (D-01) vs removing all knob art — pick whichever looks cleanest under the widget knobs.
- **Trimpot/jack row fit:** Exact widget component sizing where the 18HP row spacing differs from 14HP — match the artwork visually.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- Forge component widgets (`ForgeKnobHero/Secondary/Utility`, `ForgeTrimpot`, `ForgeJackInput/Output`, `ForgeHexBolt`) — reused unchanged except RATE/PHASE swap Utility→Secondary (D-02). Each is an SvgKnob/SvgPort with its own `_bg.svg` body, so the widget fully owns the knob visual (basis for D-01).
- Existing `mm2px(Vec(...))` + `createParamCentered`/`createInputCentered`/`createOutputCentered` idiom in the ctor — directly accepts the new anchor constants (D-07).
- `WaveformDisplay` rendering + atomic display bridge — preserved as-is; only the box geometry and internal layout constants change.

### Established Patterns
- Panel set via `setPanel(createPanel(asset::plugin(pluginInstance, "res/AnalogLFO.svg")))` — stays the same string after fresh.svg is promoted (D-03).
- Right-side hex bolt x derived from `box.size.x`, which auto-adapts to the wider 18HP panel (D-09).
- Symmetry about panel center is a hard design rule — center is now x=45.72mm (18HP), was different at 14HP.

### Integration Points
- `AnalogLFOWidget` ctor — single locus for all repositioning (display box.pos/size, every param/input/output anchor, hex bolts).
- `WaveformDisplay::phaseToX()` / `valueToY()` / column x-fractions / BPM stack / font sizes — re-tuned for 81.44×26.00mm.
- No DSP / atomic-bridge changes — purely panel + display-geometry work.

</code_context>

<specifics>
## Specific Ideas

- The new panel deliberately carries the Forge Noir identity at 18HP — single-row secondary knobs and a grouped clock section (CLK/RST/OUTPUT at x=62.01/72.88/83.74, y=119.5) are the intended reorganization.
- The bigger display glass should feel like a deliberate, polished refit of the Phase 20 look — not a stretched-as-is artifact. The user is symmetry- and spacing-focused, so column balance and consistent gaps matter (DESIGN-LANGUAGE: ~10px label-to-knob, ~14px display-to-first-label, scaled to the new layout).

</specifics>
