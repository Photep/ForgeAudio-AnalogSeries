# Phase 20.1 — UI Design Contract

> Visual and interaction contract for the 18HP Forge Noir "fresh" panel redesign. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Surface type:** This is a VCV Rack 2 module faceplate — a desktop C++ audio plugin rendered via NanoVG/SVG, NOT a web frontend. "Spacing" = mm anchor positions on the panel (LOCKED in ROADMAP). "Typography/Color" = the Forge Noir design language (carry-forward from DESIGN-LANGUAGE.md, no re-decide). This phase is a **precise repositioning + artwork-swap with zero regression** to Phase 20 display content/polish.

### Display internal layout (carry-forward, re-tuned to new box)
Phase 20 three-column rendering is **proportional** to `box.size.x`, so it re-flows into the larger 81.44×26.00mm glass automatically; re-tune constants only where balance/readability needs it (Claude's discretion per CONTEXT). Carry-forward proportions:

| Constant | Phase 20 value | Note |
|----------|----------------|------|
| Center column start | `box.size.x * 0.20` | waveform left edge |
| Center column width | `box.size.x * 0.60` | DISP-02 — trace confined to center ~60% |
| Left pill column center | `box.size.x * 0.11` | ratio / Hz / swing |
| Right pill column center | `box.size.x * 0.89` | SYNC badge / BPM stack |
| Display margins | left & right symmetric | no pill overlaps the center trace |

Display-to-content gap rhythm from DESIGN-LANGUAGE.md (~10px label-to-knob, ~14px display-to-first-label) scales to the new layout; column balance and consistent gaps matter (user is symmetry/spacing-focused).

## Typography (Forge Noir — carry-forward, no re-decide)

Panel label typography is baked into the SVG artwork as rendered paths and is **not re-set** this phase. Display readout typography is JetBrains Mono NL drawn by `WaveformDisplay` and must render with no clipping/overflow in the larger box. Re-stated here as the contract the auditor checks against.

| Role | Font | Size | Weight | Letter-spacing | Color | Usage |

| Brand | FoundationLogo (fallback Bebas Neue) | 20px @5x | — | 5px | `#e85d26` | FORGE / AUDIO |
| Module name | Bebas Neue | 20px @5x | — | 7px | `#e8e4e0` | ANALOG LFO |

| Output label | Bebas Neue | 14px @5x | — | 4px | `#e85d26` | OUTPUT |

Re-tune display font sizes against the 81.44×26.00mm box (implementer discretion) — readability over literal carry-forward — but stay within the 5–8px range so the CRT character is preserved. Hz format contract (carry-forward from Phase 20-02): 1 decimal below 10 Hz, integer at/above 10 Hz.

## Color (Forge Noir palette — carry-forward)

60/30/10 mapping for a hardware faceplate: dominant near-black panel field, secondary machined-metal/display surfaces, ember accent reserved for identity and live indicators.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (~60%) | `#0c0c0c` panel black (artwork `url(#bg)` `#0e0e11`→`#070708`) | Panel background field, display body `#030303` / `url(#screen)` |
| Secondary (~30%) | `#202024` / `url(#metal)` / `url(#metalHero)` / `url(#jack)` | Knob bodies (widget-owned), trimpots, jacks, hex bolts, display chrome |
| Accent (~10%) | `#e85d26` forge ember (+ `#d4a017` molten gold, `#f0a030` hot gold) | See reserved-for list below |
| Warm white | `#e8e4e0` / `#ece8e2` | Module name, MORPH label, knob labels |
| Destructive | not applicable | No destructive actions on this surface |

**Accent (ember/gold) reserved for:** brand wordmark (FORGE/AUDIO) + forge rune, top/bottom accent rails (`url(#rail)`), OUTPUT jack ember accent rings (r=5.30/6.10 at 83.74,119.5), display border stroke (`url(#dispStroke)`), display corner brackets, breathing border glow (DISP-05), ember waveform trace (`#ff8a4c`/`#e8612a`), phase dot core (`#f0a030`), SYNC badge, knob indicator lines, gold trimpot indicators. Never used as a fill/background.

**No new colors introduced this phase.** Zero-regression requirement (Criterion 6): ember waveform, scanlines, breathing glow, corner brackets must look identical to Phase 20.

### Panel artwork strip rule (D-01 — CRITICAL)
`res/AnalogLFO-fresh.svg` currently draws full static metal knob graphics at all 10 knob anchors (`url(#metal)` / `url(#metalHero)` body circles, inner-ring circles, machined center caps, ember/gold indicator paths). These are listed at SVG lines ~62–66 (MORPH), ~79–83, ~96–100, ~113–117, ~130–134 (secondary row), and ~140–149, ~153–162, ~166–175, ~179–188, ~192–201 (trimpots). **Strip the metal knob body / inner-ring / center / indicator art** so the widget owns the full knob — no double-rendered knob bodies (matches the proven 14HP pattern). The faint under-knob anchor rings / scallop tick marks / soft shadow MAY stay if they read well under the live widget (implementer discretion). Jacks' artwork (`url(#jack)` rings + accent ring) stays — `SvgPort` jacks sit on top.

## Copywriting Contract

This surface has no CTAs, empty states, or error dialogs (it is a hardware faceplate). The "copy" is the fixed display readout vocabulary, all carried forward from Phase 20 unchanged. Re-stated for auditor reference:

| Element | Copy / Format |
|---------|---------------|
| Primary control verbs (labels) | MORPH · CHARACTER · DRIFT · RATE · PHASE (panel-baked) |
| I/O labels | CLK · RST · OUTPUT (panel-baked) |
| Hz readout (free-run) | `{n.n} Hz` below 10 Hz, `{n} Hz` at/above 10 Hz (carry-forward 20-02) |
| Ratio readout (clocked) | ratio pill, left column |
| Swing readout (clocked) | swing pill + label, left column |
| SYNC badge | `SYNC` steady; `ACQUIRING` blinks at 2 Hz (`blinkPhase`) while locking — stays in display right column, final position for Phase 21 (D-06) |
| BPM stack (clocked) | CLK BPM + LFO BPM, right column bottom |
| Empty / error / destructive states | not applicable — no data-entry, no destructive actions on this surface |

No copy changes this phase: text content and formats are frozen; only the box geometry that hosts them changes.
