# Phase 27 — UI Design Contract

> Visual and interaction contract for the Phase 27 documentation deliverables. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Scope note (read first).** This phase builds **no interactive UI and no web app**. The only authored visual artifacts are **documentation assets**: (1) the **annotated-panel PNG** (D-08) — a real VCV Rack screenshot of the shipped `ForgeAnalogLFO` with numbered callouts baked in, paired with a numbered legend table; and (2) the **`docs/` Markdown presentation** as it renders on GitHub today and (Pages-ready) later. This contract governs the callout/annotation asset spec and the docs visual conventions, kept consistent with the shipped **Forge Noir** brand. It does NOT specify component tokens for a running app.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — C++ VCV Rack plugin; docs are GitHub-Flavored Markdown (no shadcn / React / Vite / Tailwind; shadcn gate N/A) |
| Preset | not applicable |
| Component library | none — GitHub-Flavored Markdown rendering (headings, tables, images, relative links) |
| Icon library | none — numbered callout **badges are baked into the PNG**; no icon font, no SVG icon set |
| Font | Docs prose: GitHub's native system font stack (not authorable). Baked-asset text: **JetBrains Mono** (callout numerals + data/legend), **Chakra Petch** (panel labels — already shipped in `res/AnalogLFO.svg`, appears in the screenshot; not re-authored here), **Bebas Neue** (condensed hero, only if a title plate is composited) |

**Why "Tool: none":** the shadcn initialization gate applies only to React/Next.js/Vite stacks. This is a C++17 VCV Rack 2 SDK project whose sole Phase-27 output is Markdown + one composited PNG. No design-system tooling is introduced; the registry safety gate is not applicable (see Registry Safety).

## Spacing Scale

Spacing applies to the **baked annotated-panel PNG** (callout badges + leader lines), expressed in px at the **2× export scale**. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Callout badge ring stroke; leader-line stroke width |
| sm | 8px | Gap between a control and the start of its leader line |
| md | 16px | Padding between the panel edge and composited caption/title text |
| lg | 24px | Minimum leader-line length (badge sits clear of the control) |
| xl | 32px | Minimum center-to-center separation between two adjacent callout badges |
| 2xl | 48px | Clear margin around the panel inside the composited canvas |
| 3xl | 64px | Top/bottom canvas padding of the exported annotated PNG |

Exceptions:
- **Callout badge diameter: 44px** (2× export scale). Legibility floor so numerals stay readable after GitHub down-scales the image to content width; 44 is a multiple of 4.
- **Markdown vertical rhythm is GitHub-owned, not authorable.** Paragraph, heading, list, and table spacing come from GitHub's stylesheet (and any future Pages theme). Authors control structure (heading level, blank-line separation, table shape) — not pixel spacing. Do not attempt inline HTML/CSS spacing hacks in `docs/*.md`.

## Typography

Baked-asset text sizes are px at the **2× export scale**. Markdown prose/heading sizes are GitHub-owned (values below are the effective rendered sizes for reference, not authorable tokens).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (Markdown prose, legend/reference table cells) | 16px (GitHub default) | 400 | 1.5 |
| Label (callout badge numeral — JetBrains Mono, baked) | 28px @2× | 600 | 1.0 |
| Heading (Markdown `##` section heading) | ~24px (GitHub default) | 600 | 1.25 |
| Display (Markdown `#` page title / composited panel title plate) | ~32px (GitHub default) | 600 | 1.2 |

**Weights (exactly 2):** regular **400** (prose, legend cells, leader-line-adjacent labels) and semibold **600** (Markdown headings, `**bold**` emphasis, callout numerals). Do not introduce a third weight in authored assets.

**Panel-baked brand text** (FORGE / AUDIO / ANALOG LFO wordmark + functional labels) is already rendered in the shipped `res/AnalogLFO.svg` as **Chakra Petch** path outlines and simply appears in the screenshot — it is **not** re-typeset in this phase. Do not reintroduce the purged trial FoundationLogo face (IP-01/IP-03).

## Color

Applies to the **annotated-panel PNG** (the one authored visual). Values are the shipped Forge Noir palette. GitHub's own light/dark chrome around the rendered Markdown is not controllable and is out of scope.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0c0c0c` | Near-black panel body / PNG canvas — the screenshot's own background |
| Secondary (30%) | `#e8e4e0` | Warm-white leader lines, callout numerals, and any composited legend/caption text |
| Accent (10%) | `#e85d26` | Ember-orange **callout badge ring only** |
| Destructive | not applicable | No destructive UI in this phase |

Accent reserved for: **the ember ring around each numbered callout badge — and nothing else in the annotation layer.** This deliberately mirrors the panel's ember-ring jacks so callouts read as part of the Forge Noir language rather than a foreign overlay. The gold accent `#daa520` is reserved exclusively for the shipped brand emblem already in the screenshot; do **not** use gold for callouts, leader lines, or legend text (keeps the accent singular = ember).

**Badge anatomy (prescriptive):** circular, `#0c0c0c` near-black fill, `4px` `#e85d26` ember ring, `#e8e4e0` warm-white JetBrains Mono numeral centered. Leader line: `4px` `#e8e4e0` warm white (drop to 60% opacity if it crosses busy panel art), no arrowhead, terminating `8px` from the target control.

## Copywriting Contract

Docs-native mapping of the template's interaction-copy slots. Voice is **terse reference** (D-05): controls, ranges, and behaviors stated as facts — no tutorial hand-holding.

| Element | Copy |
|---------|------|
| Primary CTA | **"Add ForgeAnalogLFO from the VCV Library"** — the install section's primary path (manual `.vcvplugin` install is the documented secondary path) |
| Empty state heading | Docs hub (`docs/index.md`) H1: **"ForgeAnalogLFO — User Manual"** |
| Empty state body | **"Analog-modeled LFO for VCV Rack 2. Choose a section below."** followed by the section link list — the hub is the landing shown when no specific section is open |
| Error state | Install troubleshooting line: **"Module not appearing? Fully quit and relaunch Rack, then re-check the plugin list."** (mirrors the stale-install / relaunch step in the build-install workflow) |
| Destructive confirmation | not applicable — no destructive doc UI action exists. The module's RESET jack is documented **neutrally as behavior** ("forces phase to 0 with 1 ms blanking"), not as a doc-level confirmable action |

**Additional copy rules (contract-level, enforced by the checker + a denylist grep):**
- **D-06 — no trademarked synth brand names anywhere in `docs/` prose.** Character voice uses generic descriptors only: "vintage transistor saw", "classic PWM square", "warm analog triangle". Denylist to keep empty across `docs/`: Moog, Minimoog, Roland, Juno, SH-101, Prophet, Oberheim, Korg (and similar).
- **D-05 — terse reference voice.** Prefer tables of facts over prose; no "let's", no step-by-step tutorials, no marketing adjectives beyond the neutral character descriptors above.
- **Changelog naming (D-09 / Pitfall 5):** use milestone names ("v1.0 Analog Series LFO" … "v1.4 Tempered"); state the manifest version is `2.0.0` for Rack 2 exactly once as context. Never relabel milestones as "2.x".

## Annotated-Panel Asset Spec (D-08)

The single authored image. **Source: a live VCV Rack 2 screenshot of the shipped, current-build `ForgeAnalogLFO`** — never the design-mockup PNGs (`forge-noir-*.png`, `panel-mockup.html`) and never the SVG export (knob art is widget-owned, absent from the SVG — Pitfall 3).

**Capture & export:**
- Verify the loaded build is current before shooting: `make install RACK_DIR=../Rack-SDK`, stale-flush if needed, fully quit + relaunch Rack, confirm `shasum` of built vs installed `plugin.dylib` matches. Run on the **main tree** (no worktree — `../Rack-SDK` must resolve; Pitfall 6).
- Capture with `screencapture -R<x,y,w,h>` (or ⌘⇧4) at **2× / retina** so numerals stay crisp after GitHub down-scales.
- **Recommended baking method: HTML-overlay + Playwright** (matches the existing `*-mockup.html` workflow, gives reproducible/tweakable callout coordinates) — place `panel-raw.png` as a CSS background, absolutely position badge + leader-line divs per the Badge anatomy above, screenshot the rendered page. **Preview markup is an acceptable manual fallback.** ImageMagick is not installed.
- Export **PNG-24**, natural panel aspect (18HP ≈ 91.44 mm × 128.5 mm — tall portrait), longest edge ~**1600px**. Save to `docs/img/panel-annotated.png`; reference with a **relative** path.

**Callout numbering (top→bottom, left→right; planner may regroup per discretion). 19 callouts, each keyed 1:1 to the legend table:**

1 Display · 2 Morph · 3 Character · 4 Drift · 5 Rate · 6 Phase Offset · 7 Morph-CV trim · 8 Character-CV trim · 9 Drift-CV trim · 10 FM-Depth trim · 11 Phase-Offset-CV trim · 12 Morph-CV jack · 13 Character-CV jack · 14 Drift-CV jack · 15 FM jack · 16 Phase-Offset-CV jack · 17 CLK jack · 18 RESET jack · 19 OUTPUT jack.

Placement note: each trimpot sits directly above the jack it attenuates; keep badges outside the panel silhouette where possible, with leader lines pointing in (see PROJECT panel geometry in 27-RESEARCH.md).

**Legend table (doubles as the control-reference table).** Single table, one row per callout number, columns:

| # | Control | Type | Range / Behavior |

Values are transcribed **verbatim** from the code tables in 27-RESEARCH.md (params/inputs/output, CV ranges, ratio table) — no paraphrase-from-memory (Pitfall 2). CV ranges come from `src/AnalogLFO.cpp`.

## Docs Presentation Conventions

Governs how `docs/*.md` reads on GitHub today and stays Pages-ready (D-02 / D-03).

- **Heading hierarchy:** one `#` H1 per file (the page/section title) · `##` for major sections · `###` for sub-points. No skipped levels. The hub `docs/index.md` H1 is the manual title and links to every section file.
- **Tables:** GitHub-Flavored Markdown pipe tables. Prefer left-aligned text columns; put numeric ranges/values in a dedicated column. Every control/IO/ratio/swing fact lives in a table, not prose (D-05).
- **Images:** relative paths only — `![Annotated panel](img/panel-annotated.png)`. No absolute repo URLs (they break locally and on a future Pages build).
- **Cross-links:** relative paths **including the `.md` extension** — `[I/O Reference](io-reference.md)` — which render in GitHub view today and under MkDocs/Docusaurus later.
- **Manifest link (D-04):** `plugin.json` gains a `manualUrl` pointing at the docs landing (e.g. `.../blob/main/docs/index.md`); existing `pluginUrl` / `authorUrl` / `sourceUrl` are kept. Public reachability completes with the Phase 28 repo-public flip — do not treat the link as live mid-Phase-27 (Pitfall 4).
- **No Pages build step this phase** — author Pages-ready structure only (D-03).
