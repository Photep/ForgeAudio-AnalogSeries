<objective>
Fix the second CI root cause (D-07): the drift-on golden uses `std::normal_distribution`, which is not
portable across libc++/libstdc++/MinGW, so Linux/Windows fail the 1e-5 tolerance. Restructure the golden
so the cross-platform CI leg replays a NEW drift-OFF + spread-OFF fixture (deterministic, small libm
epsilon), while the bit-exact drift-ON replay is gated to macOS only.

Purpose: The other half of TEST-06. This is TEST-ONLY — no shipped-audio change. The CRITICAL landmine
(RESEARCH Pitfall 1): turning drift off is NOT sufficient — `BlockDriver`'s constructor unconditionally
calls `core.setSpreadSeed(...)`, which draws from `std::normal_distribution` and perturbs the waveform
even with drift off. The fixture generation AND the drift-off replay must ALSO neutralize the spread path
(never call `setSpreadSeed` → all `*Spread` stay 0.f). Miss this and Linux/Windows stay red.
Output: a `tools/` generator + Makefile target, three drift-off fixtures, updated seeds doc, restructured
test_golden.cpp.
</objective>
